# Hyper — Bootstrap

Before any skill writes Hyper state, ensure the Hyper state root has:

```text
.hyper/
  tasks/
  archive/
```

Create missing directories as needed.

Optional files are created only by the skills that own them:

- `backlog.md` by `hyper-backlog`
- `memory.md` when a user asks to record project memory
- `retro.md` by `hyper-retro`
- `rules.md` when a user asks to record a project rule
- `recipes/` by `hyper-recipe`
- `loops/` by `hyper-iterate`
- `epics.md` by `hyper-task epic create`
- `repo.md` by `hyper-sync init`
- `jira.md` (project config) by `hyper-jira init`
- `jira.md` (task folder, before archive) by `hyper` when `jira_key` is set

`dashboard.md` is seeded at task creation per `reference/dashboard.md` and
regenerated by `hyper` after every phase return, except on `scope: code-review`
tasks (which skip dashboard generation per `reference/dashboard.md`). New task
folders contain `task.md` from creation, plus `dashboard.md` unless the task is
`scope: code-review`; other artifacts (`01-intake.md`, `02-spec.md`,
`03-technical-plan.md`, `04-execution-plan.md`, `05-execution-plan-review.md`,
`research.md`, subtask files, `checks.md`) appear as their owning phase
produces them.
